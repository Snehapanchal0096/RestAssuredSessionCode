POJO --  dependency must to add - jackson databind

***Deserialization - is process of converting json response to java object.
- it's implemented using POJO class.
- During deserialization , it's automatically maps json fields to your pojo class fields.
- once deserialized/Json response is converted into an java object , you can easily use in java.

Note - use "jackson databind" dependency is needed.


***Response Specification  - A reponse Specification is a reusable template where you store common validation for your Api responses so you don't have to repeate them in every test.

think of it like a standard checklist for response.

without it, you would write response validation repeatedly.
with Response Spec -> write once, reuse everywhere.

***Request Specification - Request specification in Rest Assured lets you define all the common request setting once and reuse them across all tests.
e.g - (base URI, base PATH, headers, auth, query param ..etc)

it helps keep your tests clean and avoid repetition.

*** validate Response structure
1. - json-schema-validator
2. create json schema file under src/test/resources - productSchema.json
3. validate API response with schema
- .body(matchesJsonSchemaInClasspath("productSchema.json")
Note - Import following static package in your script
import static io.restassured.module.jsv.jsonSchemaValidator.matchesJsonSchemaInClasspath;
